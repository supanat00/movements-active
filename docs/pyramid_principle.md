# Pyramid Principle

## When to Use This Skill
Use when you need a clear recommendation, memo, update, or executive summary.

## When Not to Use This Skill
Avoid when the task is exploratory and the conclusion is not ready yet.

## Goal
Communicate complex information efficiently by presenting the main takeaway first, followed by logical, structured supporting arguments.

## Workflow
1. Write the one-sentence answer (conclusion) first.
2. List three to five supporting arguments.
3. Check whether each argument directly supports the answer.
4. Remove overlaps and reorder the arguments logically.
5. Write the final response with the conclusion at the top.

## Validation
- The output starts with a clear conclusion.
- It provides non-overlapping supporting reasons.
- A busy reader can understand the answer without reading the whole note.

## Safety Rules
- Do not bury the main point at the bottom of the document.
- Do not mix different levels of logic in the same group.

## Output Format
A concise memo with a conclusion at the top and structured supporting points.

## References
- Internal communication guidelines.

## Stop and Escalate If
- The task does not fit the Pyramid Principle.
- Required context is missing to form a solid conclusion.
