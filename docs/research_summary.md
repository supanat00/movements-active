# Research Summary

## When to Use This Skill
Use when presented with long documents, articles, or research papers that need to be distilled into key findings.

## When Not to Use This Skill
Do not use when exact, verbatim extraction of highly specific data points or legal clauses is required without summarization.

## Goal
Provide a clear, accurate, and actionable summary of the provided materials.

## Workflow
1. Read and analyze all provided source materials thoroughly.
2. Identify the main arguments, methodologies, and conclusions.
3. Draft a high-level executive summary.
4. Extract and list the key bullet points and findings.
5. Review the summary to ensure it aligns accurately with the sources.

## Validation
- The summary accurately reflects the core message of the source material.
- Key findings are highlighted clearly.
- No hallucinations or external opinions have been added.

## Safety Rules
- Do not inject personal bias or opinions into the summary.
- Do not state hypotheses from the text as proven facts.
- Always cite the original sources for major claims.

## Output Format
A well-structured markdown document containing an executive summary, key findings (bullet points), and references.

## References
- The provided source documents or URLs.

## Stop and Escalate If
- The source materials contain highly contradictory information that cannot be resolved without domain expertise.
- The text is too technical or specialized for a general summary without specific guidance.
